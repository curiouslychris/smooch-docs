+ track:

Tracks an app event, and processes any whispers associated with that event.

Parameters

eventName

The name of the event to track. This should match a whisper created on the admin panel for your app.

Discussion

Whispers can be configured in the Smooch admin panel.

Whispers can only be fulfilled once per app user, and tracking an event after the whisper has been fulfilled will have no effect.

+ login:jwt:

Logs in a new Smooch user.

+ (void)login:(NSString *)userId jwt:(NSString *)jwt

Parameters

userId

The distinct id of the user to login. Must not be nil.
jwt

Optional jwt used to prove the origin of the login request. May be nil.

Discussion

You can either use this method to transition from logged out state to logged in, or to switch the currently logged in user to a different one.

+initWithSettings: must have been called prior to calling this method.

You may not call login while the conversation screen is shown. Doing so will result in a no-op.

+ logout

Logs out the current user.

+ (void)logout

Discussion

This method will automatically login an anonymous user after logging out the current user.

Calling this method while already in anonymous state has no effect.

You may not call logout while the conversation screen is shown. Doing so will result in a no-op.
